Vanilla Policy Gradient (with Baseline)

Posted on 2021-02-02 Edited on 2021-02-18

Openai上有一篇对Vanilla Policy Gradient进行介绍的文章

Background

上一篇介绍DQN的文章简要介绍了MDP和一些别的简单的内容，在对Policy Gradient进行介绍前，需要再明确一些概念

这次，我们用$\pi(a\mid s)$表示在$s$采取$a$的概率（这是因为Policy Gradient是一种天然的可以得到混合策略的方法）；对于一个参数$\theta$（例如一个神经网络），我们用$\pi_{\theta}$来表示由$\theta$给出的策略；$\pi^*$表示最优策略（同样，带*号的都表示optimal solution)。下面介绍的概念都是基于某一个策略$\pi$而言：

  • Value Function: $V(s)$表示从$s$这个状态出发得到的期望收益，可以用递推式$$V(s)=\sum_{s’, a} \pi(a\mid s)p(s’\mid s, a)(R(s, a) + \gamma V(s’))$$表示
  • Q Function：$Q(s, a)$表示在$s$采取$a$得到的收益，可以用递推式$$Q(s, a) = \sum_{s’}p(s’\mid s, a)(R(s, a) + \gamma V(s’))$$表示；由此可见，我们有$$V(s)=\sum_a \pi(a\mid s)Q(s,a)$$
  • **Advantage Function **：优势函数$A(s, a)$表示采取$a$比采取别的策略有多大的好处，可以直接定义为$A(s, a) = Q(s, a) - V(s)$

我们的目标同样是找到一个最好的$\theta^*$来最大化从初始状态出发得到的收益，即最大化$J(\theta)=V^{\pi_{\theta}}(s_0)$

Policy Gradient

Reinforce

通过数学推导我们可以得到$$\nabla_\theta J = \mathbb E_{\pi_\theta}\left[\nabla_{\theta}\log \pi_\theta(a\mid s)Q^{\pi_\theta}(s, a)\right]$$

在优化$\theta$时，我们只需要通过模型给出$\pi_\theta$即可，Q function可以通过在环境中使用$\pi_\theta$进行交互来得到一个对$Q^{\pi_\theta}(s, a)$估计$\hat Q(s, a)$；换而言之，我们通过$\pi_\theta$得到了一个trajectory $\tau = (s_t, a_t, r_t){t=0}^n$，那么$\hat Q(s_t, a_t)=\sum{k=t}^{n} \gamma^{k-t}r_k$

然后我们就可以通过$\nabla_\theta J \approx \sum_{t=0}^n \nabla_\theta \log \pi_\theta(a_t\mid s_t)\hat Q(s_t, a_t)$得到一个对梯度的估计$\Delta \theta$，然后用$\theta \leftarrow \theta + \alpha \Delta \theta$进行更新即可；实际操作中可以多收集几条trajectory去平均值来得到一个更好的估计

上述方法实际上就是Monte-Carlo Policy Gradient (REINFORCE)**，即通过$\pi_\theta$来收集一个trajectory**，然后得到一个对$J(\theta)$的梯度的估计来得到一个增大它的方向$\Delta \theta$，最后让$\theta$朝着这个方向前进一个步长得到新的$\pi_\theta’$

Policy Gradient with Baseline

Reinforce的原理十分简单，我们可以看到如果一个决策的能够获得收益，那么它的概率就会提升，反之则会下降；于是这就带来一个显而易见的问题，如果一个环境的reward都是正的，或者都是负的，那么就会很难收敛，甚至收敛到一个非常差的local optimum

尽管从原理上讲，即便reward能都是正的，好的策略能得到比坏的策略更高的reward，因此能够得到一个更大的梯度从而在长期获得优势，但是在实践中，一旦陷入local optimum，就很难再sample到好的策略从而跳出local optimum

因此我们希望能都解决这个 “reward都是正的” 的问题；注意到即便我们对$Q(s, a)$增添上一个函数$b(s)$是不影响梯度的，即$$\nabla_\theta J = \mathbb E_{\pi_\theta}\left[\nabla_{\theta}\log \pi_\theta(a\mid s)Q^{\pi_\theta}(s, a)\right] = \mathbb E_{\pi_\theta}\left[\nabla_{\theta}\log \pi_\theta(a\mid s)\left(Q^{\pi_\theta}(s, a) + b(s)\right)\right]，$$因而我们可以转而使用$$\nabla_\theta J = \mathbb E_{\pi_\theta}\left[\nabla_{\theta}\log \pi_\theta(a\mid s)A^{\pi_\theta}(s, a)\right]$$

这种方法也叫Policy gradient with baseline，其中baseline就是指$V(s)$，直观的理解就是通过减去一个平均收益，我们让reward有正有负了（实际上就是减小了variance）

于是，我们只需要在$\pi_\theta$的基础上多维护一个$V_\phi$，然后就能使用对Advantage function进行估计：$\hat A(s, a) = \hat Q(s, a)-V_\phi(s)$；于此同时，我们需要最小化$E_{\pi_\theta}[(V_\phi(s)-\hat V(s))^2]$来让我们的Value function给出一个尽量准确的估计

Policy Gradient Variants

通过$$\nabla_\theta J = \mathbb E_{\pi_\theta}\left[\nabla_{\theta}\log \pi_\theta(a\mid s)\left(Q^{\pi_\theta}(s, a) + b(s)\right)\right]$$ 一式我们能得到一系列的on policy的Policy Gradient算法：

  • Vanilla: $\nabla_\theta J = \mathbb E_{\pi_\theta}\left[\nabla_{\theta}\log \pi_\theta(a\mid s)\color{red}{\hat A(s, a)}\right]$并同时维护$V_\phi(s)$
    • 如果我们通过一些方法来限制步长使得模型一定获得提升则可以得到TRPO算法
  • Q Actor-Critic（通常所说的Actor-Critic: $\nabla_\theta J = \mathbb E_{\pi_\theta}\left[\nabla_{\theta}\log \pi_\theta(a\mid s)\color{red}{Q_\phi(s, a)}\right]$并同时维护$Q_\theta(s, a)$（使用TD-error更新$Q_\phi(s, a)$）
  • Advantage/TD Actor-Critic：通过将$\color{red}{Q_\phi(s,a)}$替换为$\color{red}{A_\phi(s,a)}$和TD-error实现

上诉做法本质上都是通过尽量得到一个对$Q^{\pi_\theta}(s, a)$的更好的表示来减小variance

Off policy的做法则不能通过上述方法得到（即不能直接将上面计算梯度的方法运用在过去的trajectory来得到对梯度的估计），需要一个叫importance factor的东西来计算正确(consistent)的梯度估计

GAE (Generalized Advantage Estimate)

可以引入参数$\lambda$定义广义的Advantage Function估计，同过TD-error可以定义一步估计$$\hat A^{(1)}(s_t, a_t) = \delta_t = r_t+\gamma V(s_{t+1}) - V(s_t)$$

类似的可以定义$k$步估计$$\hat A^{(k)}(s_t,a_t)=\sum_{i=0}^{k-1} \delta_{t+i} = \sum_{i=1}^{k-1} \gamma^i r_{t+i} + \gamma^kV(s_{t+k}) - V(s_t)$$；由此可见之前定义的$\hat A(s, a)$其实就是$k=\infty$的情况

可以看到$k$越大，我们的估计就约接近收集的数据，就会有较小的bias，但同时也会有较大的variance

于是我们就需要参数$\lambda$来平衡这二者的影响

定义$\text{GAE}(\gamma, \lambda)$的估计（通过添加系数使其consistent）为$$\widehat A(s_t, a_t) = (1-\lambda)(\hat A^{(1)}(s_t, a_t) + \lambda\hat A^{(2)}(s_t, a_t) + \lambda^2 \hat A^{(3)}(s_t, a_t) + \cdots)=\sum_{k=0}^{\infty}(\lambda\gamma)^k\delta_{t+k}$$

我们可以通过调整参数$\lambda$来调整，$\lambda=0$相当于用TD-error进行估计，$\lambda=1$相当于用收集到的reward进行估计

Training Tricks

Advantage Normalization

在得到一个batch的数据Advantage Function的估计后，可以对其进行Normalization

这一操作可能是consistent的（not strictly proved here)，因为$\sum_a p(a\mid s)A(s, a) = \sum_a p(a\mid s)(Q(s, a)- V(s)) = 0$

Entropy Loss (Score)

可以添加一个Entropy的贡献，即为$J$加上$\mathbb E_s [H(s)] = \mathbb E_s [-\sum_{a} p(a\mid s)\log p(a\mid s)]$，这样可以保证一定的随机性使得不会很容易的就陷入到某一个固定的局部最优策略中

DQN (Deep Q-Network)

Posted on 2021-01-28 Edited on 2021-02-18

DQN是最基本，也是被应用最广泛的RL模型之一

MDP (Markov Decision Process)

我们通常可以将一个task抽象为MDP问题。下面简单说明MDP的基本要素：

  • state：通常用来指代某一个当前任务（游戏）所在的状态，一般用observatoin指现在所观察到的信息（如当前游戏画面），而state可以直接是observation，也可以是之前一系列的observations的集合，或者别的自定义的方式
  • action：执行某一个操作；通常用action space指所有可以执行的操作的集合
  • reward：奖励，比如到达终点就给予+1的奖励；reward可以是一个基于state的函数$R(S)$，也可是基于state-actoin pair的函数$R(S, A)$
  • transition：转移，可以用$p(s’\mid s, a)$表示在$s$这个状态执行$a$这个action后转移到状态$s’$的概率
  • trajectory：在MDP中，从某一个初始状态$s_0$出发，采取了某个动作$a_0$后，根据转移概率转移到$s_1$，再采取某个动作$a_1$……，如此重复下去得到的$s_0, a_0, s_1, a_1, …$就是一个trajectory，并且这个trajectory的价值我们就认为是$R(s_0) + R(s_1) + R(s_2) + …$

因此，我们的目标就是找到一种决策的方式$\pi(s)$，表示在$s$这个state采取$\pi(s)$这个action（当然，也可以有混合策略），来最大化生成的trajectory的期望价值，可以记为$$\mathbb E\left[\sum_{k=0}^{\infty}R(s_k)\mid s_{k+1}\sim p(s_{k+1} \mid s_k,\pi(s_k))\right]$$

Q-Learning

Q-Function

我们希望得到$Q(s, a)$表示在$s$采取$a$后，得到的最大期望收益，我们可以写出转移式$Q(s, a) = \sum_{s’}p(s’\mid s, a)\left(R(s, a) + \gamma \max_{a’}{Q(s’, a’)}\right)$

  • TD error (Temporal Difference error)：等式左右两边的差，用以估计当前的Q-function还有多大的误差

如果我们得到了正确的Q-function，直接令$$\pi(s) = \arg\max_a Q(s, a)$$即可。

采用一个基于state-action的函数而不是一个单纯的基于state的函数的原因是$Q(s, a)$可以做到model-free，也就是在不知道转移概率$p(s’ \mid s, a)$时仍然可以学习，而只基于state的函数不行

在实际问题中，一个重要的困难在于state的数量往往是很大的，甚至是无穷的（例如state可以是一个连续变量），因此我们可以用一个神经网络来拟合Q-function，并且在训练时通过收集到的$s, a, s’, r(s, a)$通过转移方程来更新

On-Policy 与 Off-Policy

On-policy指的是每次都只用最新的一次trajectory来更新，而off-policy则可以将之前的trajectory中收集到的某一个转移$s, a, s’, r(s, a)$存储到memory中，并每次从memory中提出一个batch的转移来更新

Off-policy的方法具有明显的更高的数据使用率，但是缺陷是memory中的数据分布与当前的policy并不符合，因此会出现一些问题（如over estimate，对未访问的状态往往具有过高的估计，可以想象，对一个具有noise的分布取max得到的值一定大于实际上的平均值）

Exploration

RL中的一个重要的问题就是exploration，也就是在训练的时候采取什么决策方式；简单每次采用使得Q-function最大的action的方法很容易陷入一些局部的最优解；一种简单的解决方式为$\epsilon$-greedy，每次有$\epsilon$的概率选择一个随机的解，其余按照当前的Q-function贪心选择最优解

在实际中，$\epsilon$-greedy已经是一个优秀的策略；基于exploration策略的研究是RL的一个方向，比较典型的有 Count-Based DQN，是通过给较少经过的状态更高的权值来达到explore的效果

CNN (Convolutional Neural Network)

CNN（卷积神经网络）是一种以卷积层为核心的神经网络。一般认为，卷积层是一种良好的从二维图像中提取特征（feature）的结构，因此CNN被广泛应用于图像识别领域（CV，computer vision and pattern recognition），从一开始的Alex-Net（开启了神经网络的时代）到Res-Net（the first very deep network)。一个简单的CNN可以由以下单位组成：

Convolutional Layer

下面简单说明卷积层的常用参数：

  • in_channels: 输入的channel的数量（可以认为是有多少张图片叠在一起输入，例如如果输入的是RGB图片娜美这一参数就是3）
  • out_channels: 输出的channel的数量，可以理解为从之前的输入中提取出多少不同的feature
  • size：卷积核的大小，即对多大的一个子矩形进行聚合
  • stride：步长，每多少行多少列进行一次卷积操作
  • padding：在图像的边缘补充的空白，一般要么为0，要么设置为size的一半（这样可以让新的图像的大小恰好按照size进行划分）

Pooling Layer

池化层，常用的就是对一个子矩形求最大值，目的是为了缩小图片的大小以减少参数

Fully-Connected Layer

全连接层，本质就是一个矩阵；在最后一层往往使用全连接层来实现分类，而前面的所有结构都可以视为提取特征

Activation Function

激活函数通常是一个非线性函数，在实践中最常见的为ReLU(Rectified Linear Unit)。卷积层+激活函数、全连接层+激活函数（常称为MLP）都被视为是提取feature的方式。

Why Conda?

conda 可以用于管理软件包（package），尤其是python的软件包，可以在conda的不同环境（environment）下安装每个环境所需要的package，这样就不会互相干扰，并且易于管理。例如，如果要同时安装python2和python3，就可以在conda下开两个环境分别安装，在调用时启动对应的环境即可。
